# Traceability matrix

This matrix reflects the **real current state** of traceability and executable coverage in the repository after introducing:

- canonical functional scenarios in `docs/features/`, and
- lightweight JUnit traceability in backend tests via `@SpecificationRef`.

The intended chain remains:

**Requirement -> Scenario (`.feature`) -> Test -> Code**

> Note: the `.feature` files are not executed with Cucumber. They are maintained as canonical functional specification and traceability anchors.

## Status legend

- **NOT STARTED**: there is no executable test that reasonably demonstrates the scenario.
- **PARTIAL**: there is at least one relevant executable test, but important layers or assertions are still missing.
- **COVERED**: the scenario is adequately covered at the layers needed by its current specification.

## Interpretation notes

- Backend tests annotated with `@SpecificationRef` provide the strongest current traceability for the backend side of the matrix.
- Frontend entries are based on real existing Vitest/RTL tests, including co-localized tests next to pages and components.
- A scenario is only marked as **COVERED** when the currently existing tests match the scenario text closely enough without inventing missing behavior.
- This document is intentionally conservative so it can be reused honestly in the TFG memory and in later implementation phases.
